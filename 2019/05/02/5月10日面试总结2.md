#### 2.HTTP缓存机制及原理?
http 请求分为强制缓存和对比缓存
* 在客户端第一次请求数据时,此时缓存数据库中没有对应缓存数据,需要请求服务器,服务器返回后,将数据存到缓存数据库中

    ![强制缓存流程图](img/2.png)
    ![xxxx](img/1.png)

*  HTTP缓存有多种规则，根据是否需要重新向服务器发起请求来分类，我将其分为两大类(强制缓存，对比缓存)

> 强制缓存:浏览器第一种情况请求数据是从`缓存数据`库中请求,如果有且未失效则返回数据.第二种情况请求缓存数据库没有数据或者数据已经失效,则像服务器发送请求,服务器会返回数据以及`缓存规则`
浏览器再将服务器返回的数据和缓存规则存到缓存浏览器中.
> 对比缓存:第一种情况是浏览器从 `缓存数据库`中获取缓存数据的标识,并且请求服务器缓存标识对应的数据是否生效,如果未失效,则通知客户端缓存未失效.然后可以从缓存数据库获取数据,
第二种情况也是从`缓存数据库`中获取缓存数据的标识,然后请求服务器验证缓存标识对应的缓存数据是否失效,如果已经失效,服务器则会返回最新的数据和缓存规则,然后客户端再将缓存规则和最新数据存入缓存系统.

#### 强制缓存的规则的有限性大于对比缓存规则

###### 原理大概是浏览器和服务器通过请求头和请求体进行交互过程通过一些规则是否设置存储方便优化网络加载速度的处理.
#### 3.简单实现一个js多线程?
JS是一种单线程语言，即使是一些异步的事件也是在JS的主线程上运行的像setTimeout、ajax的异步请求，或者是dom元素的一些事件，都是在JS主线程执行的，这些操作并没有在浏览器中开辟新的线程去执行，而是当这些异步操作被操作时或者是被触发时才进入事件队列，然后在JS主线程中开始运行。
单线程的问题,如果说一个脚本语言在执行时，其中某一块的功能在执行时耗费了大量的时间，那么就会造成阻塞。用户体验非常不好.
>**实现**:js为我们提供一了一个worker的类,他的作用就是为了解决阻塞现象.当我们使用这个类的时候,他就会向浏览器申请一个新的线程.这个线程就用来单独执行一个js文件.
``
var worker=new Worker(js文件路径)
``
那么这个线程就会申请一个线程来执行这个js文件
下面列举几个常见的用法:
```
//postMessage(msg);
//postMessage方法把在新线程执行的结果发送到浏览器的js引擎线程里
worker.onmessage = function(){
    //获取在新线程中执行的js文件发送的数据 用event.data接收数据
    console.log( event.data )
};
setTimeout( function(){
    worker.terminate();
    //terminate方法用于关闭worker线程
},2000)

setTimeout( function(){
    worker = new Worker("js/test22.js");
    //再次开启worker线程
},3000)
```
然后再新线程中随时用postMesssage()方法可以像主线程发送一些数据,主线程使用worker的onMessage事件来接收这些数据就实现了js的单线程和多线程之间的数据交互.

#### 5.设置 margin 的上下和设置宽高无效，其他有效的元素有哪些？
回答这个题首先得明白为啥设置margin的上下和设置宽高会无效,我们回顾一下行内元素的特点就不难发现:
a.和其他元素都在一行上,
b.高和外边距无法改变
c.宽度就是他文字或者图片的宽度,无法改变,内联元素只能容纳文本和其他内联元素
d.设置margin和padding只有左右有效,上下无效.
**其他有效元素**是行高:lineHeight可以改变,其次是margin和padding的左右可以改变,既是margin-left和margin-right以及padding-left和padding-right.
#### 6.代码执行后会发现 li 标签之间有间距，如何修改？

简单上一下代码;
```
<style>
        ul{
            width: 300px;
            height: 50px;
        }
        li{
            width: 50px;
            height: 50px;
            background: blue;
            display: inline-block;
        }
    </style>
<ul>
        <li></li>
        <li></li>
    </ul>
```
上面的代码我们用diplay属性将li标签转为行内块元素,那么他将会一行显示.行内元素换行和空格以及tab键等会是标签之间产生空隙,
**解决办法** a. 去掉元素之间的空格以及换行等
```
<ul>
        <li></li><li></li>
</ul>
```
b.给标签的父元素加上font-size:0;
```
ul {
font-size:0;
}
```
